Delaunay三角网格动态添加删除点

发表于 2020-06-27 | 分类于 算法设计

引

delaunay三角剖分有很成熟的算法，，但是算法大都是给定一个点集，输出三角剖分后的结果。但是再点集是动态增加删除的场景，每次如果增加一个点都重新对全局做剖分，明显效率非常的浪费，本文主要探讨在已剖分好的三角网格上，动态增加点或者删除点，只通过局部调整保持全局合法的三角剖分。

背景算法介绍

常用的Delaunay算法有翻边算法、逐点插入算法、分割合并算法、Bowyer-Watson算法等，我们这里基于计算几何：算法与应用中介绍的随机增量式算法。先介绍原始算法：

  1. 先用一个足够大的三角形把整个点集都包围起来，如下图：
  2. 按随机次序逐一插入各点，整个过程中，都要维护并更新一个与当前点集对应的Delaunay三角剖分。考虑引入点$p_r$的情况。
    2.1 确定$p_r$落在某个三角形内，将$p_r$与该三角形的三个顶点相连形成新的三个三角形
    2.2 如果$p_r$碰巧落在某条边$e$上，找到$e$关联的两个三角形，然后将$p_r$与对顶的两个顶点相连，生成4个新三角形
  3. 引入点$p_r$后原来某些边可能不再合法，对每一条可能的非法边调用子函数LegalizeEdge，如下图，插入$p_r$后，边$p_ip_j$可能是非法边，找到共边三角形的另一个顶点$p_k$，若$p_k$在三角形$p_rp_ip_j$的外接圆内，说明边$p_ip_j$非法，断开$p_ip_j$，链接$p_rp_k$，但$p_jp_k$和$p_ip_k$有可能又变成非法边，对他们递归调用LegalizeEdge。
  4. 删除第一步引入的超大三角形顶点及其相连的边

删除一个点

比起新增点，删除点要麻烦的多。
第一个idea，把要删除点影响的三角形全部删除，再对相邻点的点集重新做delaunay生成局部补丁，再将补丁补回整体网格上：

这个idea有个明显的bug，如果待删除点和相邻点构成的多边形不是凸的，局部补丁就会冗余出三角形，如下，橘色三角形会生成两次：

而且，新生成的边仍然可能是非法边，比如EF边可能就是非法边
第二个idea，把要删除点看成是移动到一个相邻点而湮灭掉了，这样带动边一起湮灭，如果选取合适的相邻点，可以避免上面的生成冗余三角形的问题。
但是这个idea也是有问题的，很容易就构造出一个反例，无论移动到那个相邻点都会生成非法三角形，如下图，绿线是点H删除后合理的边，点H无论是移动到BCDEJK任一点都无法覆盖到绿线：

因此有回到第一个idea，进行改进：

  1. 生成待删除点相关三角形的并集多边形，即图remove2的多边形ARBCDEF
  2. 删除所有的待删除点相关三角形
  3. 待删除点相邻点组成局部点集进行delaunay剖分生成局部补丁
  4. 遍历局部补丁的所有三角形，删除不在多边形内的三角形
  5. 将过滤后的局部补丁补回全体三角网格
  6. 对并集多边形的边做LegalizeEdge

具体代码实现见delaunayCode，代码中附带一个可视化测试工具，右键增加一个点，左键删除一个点，增加点似乎不太有bug，但是删除点在点很密集的情况下仍然会触发bug

可能的原因是：

  1. 算法还是不完备，是否有可能局部delaunay三角剖分不包含并集多边形的某条边
  2. 三边形是否在并集多边形内，是通过判断三角形的重心是否在并集多边形内判断的，点在多边形内用的是cross-ray算法，这个算法在边界情况有问题，比如下图中的M点和Z点都会被判断在多边形外，因为这两个测试点和两个顶点在同一水平线上。
