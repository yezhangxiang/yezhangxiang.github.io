参加了个算法比赛

发表于 2015-04-26 | 分类于 算法设计

今年无线的算法日还包括一个算法大赛，可惜我已经不在无线了，没有参赛资格，老大心血来潮把无线算法大赛的题目拿来办小组算法大赛，拿两个工作日的晚上加班做了个结果，白天还不好意思做题，囧～

题目

题目其实就是一个简化版的自动站点规划：给定一个栅格化的矩形区域的用户分布，在规定时间内（5 min）在该区域内加M个全向站，使得得分 $Fitness$ 尽可能高。具体描述如下：

  • 1.数据集描述
    给定规划区域和用户分布：
    规划区域为矩形区域，该矩形区域由X×Y个正方形栅格组成，米格正方形栅格的边长为10m。
    用户分布同样以X*Y矩形形式给出，矩阵的元素值为大于等于0的整数，对应该栅格中的用户总数。
    左下角为原点(0, 0)。

  • 2.任务描述
    对于给定的规划区域范围和用户分布，及站点数M，求：
    规划并输出全向站点的坐标，以及每个站点的导频信道RS发射功率，算法评价指标（见下节）尽可能高。
    备注：

    • 1）规划站点坐标必须落在规划区域范围内
    • 2）不考虑全向站点站高；
    • 3）站点RS发射功率必须在9.0dBm～15dBm以内。

  • 3.量化的评价标准

    • a.RSRP和SINR的计算
      对于给定的位置(x, y)接受到站点i(Xbi, Ybi)发射信号的RSRP，单位dBm，RSRP简化由如下公式计算：$$\begin{equation} RSRP(x, y, indexBi) = Pbi-[A+B*log_{10}(\sqrt{(Xbi-x)^2+(Ybi-y)^2)}] \end{equation}$$ 其中$\sqrt{(Xbi-x)^2+(Ybi-y)^2}$为给定位置到站点的距离，A， B为可配置参数。
      对于给定位置(x, y)接收到周围站点发射的RSRP信号列表为：$RSRP_1, RSRP_2, ..., RSRP_n$，用公式$P=10^{(\frac{RSRP}{10}-3)}$将其单位有dBm转换为w，记为$P_1, P_2, ..., P_n$，其中PP最大值为$ P_m = \max( P_1, P_2,…,P_n ) $ 。对于给定位置(x, y)的SINR，单位dB，SINR简化计算如下：$ SINR(x, y) = 10log_{10}(\frac{P_m}{N+\sum_{i=1,i\neq m}^{n}P_i}) $ 其中$N=3e-16=3*10^{-16}$
    • b.站点覆盖的栅格和门限
      对于某栅格i接收到周围站点发射的RSRP信号列表为：$RSRP_1, RSRP_2, …, RSRP_n$，其中RSRP最大值为$RSRP_j = \max( RSRP_1, RSRP_2,…,RSRP_j,…,RSRP_n )$,则该栅格归属于站点j.
    • c.算法评价指标——最大化Fitness：$$\begin{equation} Fitness = w*Fitness_{area} + (1-w)*Fitness_{ue},w\in[0, 1] \end{equation}$$ 区域覆盖指标评价为：$$\begin{equation} Fitness_{aera} = \frac{\sum_iF_{grid}(i)}{N_{grid}} \end{equation}$$ 其中:$$\begin{equation} F_{grid}(i)=\left\{ \begin{array}{c} 1 & if \ \ RSRP_j(i)>=T_{RSRP} \ \ and \ \ SINR(i)>=T_{SINR}\\ 0 & others \end{array} \right. \end{equation}$$ 评价用户覆盖指标$Fitness_{ue}$:$$\begin{equation} Fitness_{ue} = \frac{\sum_jFBNUM(j)}{N_{ue}} \end{equation}$$ 对于某个栅格i内用户数为$UENUM_i$，则该栅格是否满足评价指标决定该栅格内所有用户是否满足评价指标，即 $F_{grid}(i)*UENUM_i$
      对于某一站点j其覆盖的满足指标的用户数为：$$\begin{equation} FBNUM(j) = \min(\sum_{i\in S_j}(F_{gird}(i)*UENUM_i), T_{UENUM}) \end{equation}$$

设计思路

比赛委员会给的提示是，解决此类问题业界一般使用寻优算法，但是根据我之前的经验，这个问题直接使用寻优算法效果肯定不好，维度太高，解空间的对称性严重（把一个解中的两个小区的所有参数对调就得到效果完全一样的另一个解），先简单粗暴的直接用matlab自带的ga算法尝试了一下，果然直接上寻优算法效果很糟糕。
于是思路为，找到一个不错的初始解，根据初始解限制每个维度的寻优范围（规避对称性），并给维度分组（降维）。具体分三步：

Phase 1. 解决覆盖问题

只考虑RSRP和SINR的话，最好的站点分布当然是正规的蜂窝网络：

但是如何在给定的矩形内按照正六边形分布M个点，我没想出好的解决办法，把这个问题化简为在一个矩形内平均分布M个点，谷歌了半天居然没有找到相关的资料，这应该是个很常见的问题啊。倒是搜索出来很多如何在一个球面上平均分布若干个点的研究。
后来尝试了一下直接对栅格做聚类算法，效果倒是出乎意料的好，效率很高，而且结果基本就是voronoi图，如果全向站的功率一样的话，他们的覆盖就应该是个voronoi图。把一个正方形区域分为4，5，6，7个区域，结果如下：

Phase 2. 根据话务地图修正参数

在第一步的基础上，计算每个小区的话务量，如果当前小区的话务量超出了门限 $T_{UENUM}$则记为问题小区，若两个小区的存在覆盖栅格为相邻栅格，则定义为邻区，每次修正，将问题小区的覆盖栅格分一层给非问题邻区。如下图，小区5为当前小区，邻区1，3，4，6为非问题邻区，小区2，7为问题小区，一次修正话务后结果如下：

相应的，功率和站址也要做出相应的修改，以使得覆盖范围大致符合修正的结果。
下图左图是kmeans的结果，标题中的数字是对应的Fitness；中图是话务修正的结果，右图是根据话务修正后的参数真实计算出来的小区覆盖，和我们话务修正的还是比较像的。
kmeans算法有一定的随机性，比如下图都是把正方形区域分为5个区域，明显下面一个更符合均匀的感觉，可以设计kmeans函数中的’Replicates’参数，多做几次聚类，选最好的一次。

Phase 3. 寻优算法

有了上面两步给出比较好的初始值后，应用粒子群算法来做寻优（用粒子群只是因为实现比较简单╮(╯▽╰)╭）

  • 限制每个小区的位置只能在覆盖区域内调整
  • 小区分组，优化区域有X×Y个栅格，M个小区，每个小区在所有栅格上都可以计算出一个RSRP，可以把一个小区看作是一个维度为X×Y的点，分组问题就化简为对X×Y维空间中的M个点做聚类算法。
  • 每次小组寻优后都判断剩下的时间够不够一次小组寻优，不够就退出且输出结果。

实现技巧

预先计算好所有路损

RSRP简化由如下公式计算：
$$\begin{equation} RSRP(x, y, indexBi) = Pbi-[A+B*log_{10}(\sqrt{(Xbi-x)^2+(Ybi-y)^2)}] \end{equation}$$
可将所有的$ [A+B*log_{10}(\sqrt{(Xbi-x)^2+(Ybi-y)^2)}]$ 算好放内存中，以后每次使用只要读取即可。

寻优过程中计算范围

寻优过程中之需要计算影响范围内的栅格电平，其中影响范围定义为每组调整小区的外接矩形外扩3/4个影响半径，而影响半径定义为调整小区到零层基站的最大值。

改进方向

是否可以在聚类算法中直接加入话务限制，这样就把Phase 2在Phase 1 中完成了，而且逻辑简单，效率也快不少，而且聚类的结果和voronoi图相似，也就是直接和仿真的结果相似，就可以直接根据覆盖的情况计算出小区的参数，连Phase 3都省了，不过没什么时间继续尝试了。

结果
