在系统自带的头文件出现error C2144

发表于 2014-11-09 | 分类于 编程语言

今天Debug找一个问题花了一个多小时，记录一下：
开发环境是VS2010，写了一个socket的服务器，代码大都是晚上下载的，学习一下socket编程，但是当我加了个头文件，把函数的声明放到头文件里，并且把c风格改为c++风格（printf改为cout，char *改为string）build的时候报错：

excpt.h(29): error C2144: syntax error : ‘int’ should be preceded by ‘;’

双击跳到问题语句，是最普通的

中的extern "C" {.

看到编译器给的结果很困惑：

  • 我根本没有用到excpt.h这个头文件
  • extern "C" {为什么是int

第一个问题应该是因为我包含的某个头文件包含了excpt.h，但是问题出在哪里呢？

刚开始怀疑是excpt.h的问题，但这是系统自带的头文件，应该不会少了;包含excpt.h的头文件也是系统自带的头文件，也应该不会少了;

后来怀疑是因为在自己写的头文件里把c风格改成c++风格，注释后仍有问题

google关键字error C2144 extern “C” 找到这个网址：

The problem may be in another file that you include before sal.h in your cpp file. Files are included in a cpp file as if they are one huge cpp file. So, if you forget a ‘}’ or endif in an include file, the error may be detected a bit later, in another include file of the same cpp main file.

Forget a } somewhere, or a semicolumn after a class declaration and the compiler is really lost !

突然顿悟是因为在自己的头文件里最后一行的函数声明少了”;”号，一看果然如此。一个逗号花了一个小时！

对编译原理了解深一些其实这个bug很容易解决，之前在《程序员的自我修养》里看过的，都忘了。。。

总结 编译报错error C2144，且问题出现在自带的头文件时，很可能是用户的某个头文件最后一行少写了逗号。

遗留 编译器报错说 extern "C" {是int型，还没搞明白是为什么
